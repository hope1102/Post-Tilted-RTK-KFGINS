
# 倾斜RTK实现方案

## 一、系统架构概述

倾斜RTK在KFGINS的基础上新增**三个核心模块**，形成完整的倾斜测量解决方案：

```
┌─────────────────────────────────────────────────────────┐
│                      主程序入口                           │
│                    (kf_gins.cpp 扩展)                     │
└──────────────────┬──────────────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────────────┐
│                 倾斜RTK状态机                             │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │ 初始化  │→│零偏估计 │→│航向对准 │→│ 测量模式 │       │
│  │  阶段   │  │  阶段   │  │  阶段   │  │(ZUPT+RTK)│      │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘       │
└──────────────────┬──────────────────────────────────────┘
                   │
     ┌─────────────┼─────────────┐
     ▼             ▼             ▼
┌─────────┐  ┌─────────┐  ┌─────────────────┐
│ZUPT检测器│  │航向对准 │  │ZUPT量测更新     │
│(双重条件)│  │  模块   │  │(杆臂速度观测)   │
└─────────┘  └─────────┘  └─────────────────┘
```

## 二、新增配置文件设计

### 2.1 配置文件结构

新增 `config/tilt-rtk.yaml`，完全复用KFGINS的YAML格式，同时新增以下配置节：

```yaml
# ============ 倾斜RTK专用配置 ============

# 工作流程控制
workflow:
  # 零偏估计阶段：静止时长 [秒]
  gyro_bias_est_time: 2.0
  
  # 航向对准阶段：摆动对准时长 [秒]
  heading_align_time: 3.0
  
  # 航向对准有效性的GNSS速度阈值 [m/s]
  # GNSS位置变化超过此阈值才认为轨迹有效
  gnss_pos_threshold: 0.01

# ZUPT检测器配置
zupt:
  # IMU静止检测（加速度方差）
  acc_threshold: 0.05          # [m/s²] — 静止时加速度方差阈值
  acc_window: 10              # 滑动窗口大小 [样本数]
  
  # IMU静止检测（角速度阈值）
  gyro_threshold: 0.1         # [deg/s] — 静止时角速度阈值
  
  # GNSS静止检测
  gnss_vel_threshold: 0.02   # [m/s] — GNSS速度阈值
  gnss_window: 5             # 滑动窗口大小 [样本数]
  
  # 双重条件同时满足才触发ZUPT
  require_both_conditions: true
  
  # ZUPT启用后的最小持续时间 [秒]
  # 避免ZUPT频繁启停
  min_zupt_duration: 0.5

# 杆臂配置
leverarm:
  # IMU到杆尖的杆臂 [前, 右, 下] [m]
  # 即IMU中心到RTK杆尖触地点的距离
  tip: [0.0, 0.0, 1.8]
  
  # IMU到GNSS天线的杆臂 [前, 右, 下] [m]
  # 复用原有 antlever 配置
  antenna: [0.136, -0.301, -0.184]
```

## 三、各模块详细设计

### 3.1 倾斜RTK状态机

**状态定义**（5个状态，依次推进）：

| 状态ID | 状态名称 | 进入条件 | 退出条件 | 主要行为 |
|--------|---------|---------|---------|---------|
| `INIT` | 初始化 | 系统启动 | 收到第一帧GNSS | 等待初始位置收敛 |
| `GYRO_BIAS_EST` | 零偏估计 | 进入后立即 | 静止满2秒 | INS机械编排 + 累积陀螺数据平均 |
| `HEADING_ALIGN` | 航向对准 | 零偏估计完成 | 摆动满3秒 | INS+RTK双轨迹提取 → 夹角计算 |
| `MEASURE` | 测量模式 | 航向对准完成 | 手动停止 | 正常KFGINS + ZUPT检测 + ZUPT更新 |
| `ERROR` | 错误状态 | 任何阶段失败 | 重置 | 输出错误日志，等待重启 |

**状态转换图**：

```
INIT → GYRO_BIAS_EST → HEADING_ALIGN → MEASURE
  ↓         ↓               ↓             ↓
ERROR     ERROR          ERROR         ERROR
```

**自动阶段检测逻辑**：

- 状态机根据**实时时间戳**自动判断阶段切换
- 不需要外部干预，全自动运行
- 每个阶段的时长由配置文件设定

### 3.2 ZUPT检测器（双重条件）

**设计原理**：ZUPT触发必须同时满足IMU静止条件和GNSS静止条件，提高可靠性。

#### 条件一：IMU静止检测

**双指标检测**（两者都满足才算静止）：

| 指标 | 计算方法 | 阈值 |
|------|---------|------|
| **加速度方差** | 滑动窗口内三轴加速度的方差之和 | < 0.05 m/s² |
| **角速度幅值** | 滑动窗口内角速度矢量的模均值 | < 0.1 °/s |

**数学表达**：

```
静止条件 = (Var(acc_N) + Var(acc_E) + Var(acc_D) < acc_threshold²)
           AND
           (mean(|ω|) < gyro_threshold)

其中窗口大小 = acc_window / imudatarate
```

#### 条件二：GNSS静止检测

**位置变化率检测**：

```
静止条件 = (平均GNSS速度 < gnss_vel_threshold)
         其中平均GNSS速度 = mean(|Δpos / Δt|) over gnss_window
```

**为什么需要双重条件**：

| 条件 | 能检测 | 不能检测 |
|------|--------|---------|
| 仅IMU | 机体静止 | GNSS信号跳变 |
| 仅GNSS | GNSS跳变 | IMU高频振动 |
| **双重** | **两者都能检测** | — |

### 3.3 航向对准模块

**核心原理**：利用INS积分推算的轨迹和RTK-GGA的轨迹在水平面上的**夹角**，来估计初始航向误差。

**工作流程**：

```
阶段2开始
    │
    ├─► INS机械编排：每帧输出 INS 轨迹点 (lat_ins, lon_ins)
    │
    ├─► GNSS解析：每帧输出 RTK 轨迹点 (lat_rtk, lon_rtk)
    │
    ├─► 轨迹采集窗口（摆动3秒内）：
    │     收集INS和RTK的轨迹序列
    │
    └─► 夹角计算（摆动3秒结束后）：
          ├─ 对INS轨迹做线性回归 → 获得 INS 航向角 θ_ins
          ├─ 对RTK轨迹做线性回归 → 获得 RTK 航向角 θ_rtk
          ├─ 计算航向误差 Δθ = θ_ins - θ_rtk
          └─ 将 Δθ 作为初始航向角补偿加到 initatt[2]
```

**夹角计算方法**（线性回归法）：

```
θ_ins = arctan2(Σ(y-ȳ)(x-x̄), Σ(x-x̄)²)  # INS轨迹方向
θ_rtk = arctan2(Σ(y-ȳ)(x-x̄), Σ(x-x̄)²)  # RTK轨迹方向
Δθ    = θ_ins - θ_rtk                     # 航向误差
```

**有效性判据**：

| 条件 | 说明 |
|------|------|
| GNSS轨迹长度 > 0.1m | 确保GNSS有足够运动 |
| INS轨迹与RTK轨迹相关系数 > 0.8 | 确保两者反映同一运动 |
| GNSS位置精度（GST的sdx, sdy）< 0.05m | 确保RTK精度足够 |

**初始航向补偿**：

```
补偿后航向 = initatt.yaw + Δθ
```

### 3.4 ZUPT Kalman滤波器量测更新

**当ZUPT触发时**，将杆尖速度为零作为虚拟观测引入滤波器。

#### 观测模型

**观测向量**：

```
z_zupt = v_T (杆尖速度) = 0 (因为杆尖触地静止)
```

**观测矩阵**（论文公式19-21）：

```
        |  0_3x3    I_3     H_att    H_bias    H_scale  |
H_zupt = |                                           |
        |  0_3x3    0_3     H2       0_3       0_3    |
```

其中：
- `H_att` = 姿态误差对杆尖速度的偏导数
- `H_bias` = 陀螺零偏对杆尖速度的偏导数
- `H2` = 包含杆臂矢量的复杂偏导数（详见论文）

#### 量测噪声

```
R_zupt = diag([0.01, 0.01, 0.01])  # [m/s²]
```

使用较小的观测噪声，使滤波器高度信任ZUPT约束。

### 3.5 杆臂+ZUPT协同观测模型

这是倾斜RTK的核心创新点。

**物理意义**：

当杆身倾斜时，IMU在旋转，即使杆尖静止：

```
v_T = v_I + ω × l  (杆尖速度 = IMU速度 + 杆臂旋转产生的速度)
```

**其中**：
- `v_I` = IMU中心速度（由INS推算）
- `ω` = 机体角速度（由陀螺测量）
- `l` = 杆臂向量（IMU到杆尖）

**当ZUPT触发时**，`v_T = 0`，因此：

```
0 = v_I + ω × l
```

这个方程将 `v_I`、`ω`、`l` 联系起来，形成一个强有力的约束。

**量测更新流程**：

```
每帧检测ZUPT条件
    │
    ├─ 条件满足？ → 计算 H_zupt 矩阵
    │                  │
    │                  ├─ H_pos: 位置误差的雅可比
    │                  ├─ H_vel: 速度误差的雅可比
    │                  ├─ H_att: 姿态误差的雅可比
    │                  └─ H_bias: 零偏误差的雅可比
    │
    ├─ 计算新息：dz = v_T (杆尖速度) = 0
    │
    ├─ 设定量测噪声：R_zupt
    │
    └─ 执行EKF更新：dx, Cov
```

## 七、关键参数配置建议

基于论文实验数据，推荐以下参数：

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 零偏估计时长 | 2秒 | 平衡精度与启动速度 |
| 航向对准时长 | 3秒 | 确保杆身有明显摆动 |
| IMU加速度阈值 | 0.05 m/s² | MEMS IMU典型噪声水平 |
| IMU角速度阈值 | 0.1 °/s | 静止时陀螺输出接近零 |
| GNSS速度阈值 | 0.02 m/s | RTK厘米级精度 |
| ZUPT量测噪声 | 0.01 m/s | 信任ZUPT约束 |
| 杆臂长度 | 1.8 m（默认） | RTK杆标准长度 |

## 八、预期效果

根据论文实验数据，倾斜RTK相比KFGINS的提升：

| 指标 | KFGINS（无倾斜补偿） | 倾斜RTK（有ZUPT） |
|------|-------------------|------------------|
| ≤2cm位置精度比率 | ~70% | **~90%** |
| 大角度（>45°）表现 | 差（航向漂移） | **显著改善** |
| 航向精度 | 依赖初始对准 | **航向实时校正** |
| 零偏估计 | 配置固定值 | **自动估计** |
